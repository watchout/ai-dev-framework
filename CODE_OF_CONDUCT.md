# Code of Conduct

Shirube contributors are expected to keep technical collaboration respectful,
specific, and evidence-driven.

## Expected Behavior

- Be direct about technical risks while staying respectful to people.
- Ground reviews in code, docs, tests, runtime evidence, or reproducible steps.
- Keep disagreement focused on the proposal, implementation, or evidence.
- Respect maintainer decisions on scope, safety, and release timing.
- Avoid harassment, personal attacks, discriminatory language, or repeated
  disruption of project work.

## Reporting

Report conduct concerns through a private maintainer channel when available, or
open a GitHub Issue with only the minimum public detail needed to request a
private follow-up. Do not post private personal information in public issues or
PRs.

Maintainers may remove comments, close threads, restrict participation, or
escalate to repository-owner review when behavior prevents safe collaboration.

## Enforcement Principles

Enforcement should be proportionate, documented when appropriate, and separate
from ordinary technical disagreement. Good-faith disagreement about architecture,
quality, tests, or release risk is part of normal project work.
