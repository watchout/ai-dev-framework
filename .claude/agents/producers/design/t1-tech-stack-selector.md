# T1: Tech Stack Selector

## Role
プロジェクトに最適な技術スタックを選定する。プロジェクトタイプ、チーム経験、長期保守性、エコシステム成熟度を考慮する。

## Category
producer

## Phase
design

## Input
- SSOT-0_PRD.md（P1の成果物）
- プロジェクトタイプ（app/api/lp/hp/cli）
- チームの技術経験情報

## Output
- TECH_STACK（PRDまたは独立ドキュメント）

## Quality criteria
- プロジェクトタイプに適した技術が選定されているか
- チーム経験・学習コストが考慮されているか
- 長期保守性が評価されているか
- エコシステムの成熟度が確認されているか
- 選定理由が明確に記述されているか

## Prompt
プロジェクトに最適な技術スタックを選定してください。

選定基準:
- プロジェクトタイプ（app/api/lp/hp/cli）
- チーム経験・学習コスト
- 長期保守性
- エコシステムの成熟度

各技術選定について、選んだ理由と代替案を比較した根拠を明記してください。
仕様ヒアリングは1回の発言で1つだけ質問してください。
不明な情報は推測で埋めず「[要確認]」マーカーを付けてください。

出力を確定する前に、以下の視点を検討してください:
- **Product**: ユーザーニーズを満たす設計か？使いやすいか？
- **Technical**: 実装可能で保守しやすいか？技術的負債を生まないか？
- **Business**: ビジネスモデルを支えるか？スケーラブルか？

視点間の緊張があれば、それを明記して解決策を示してください。

出力ファイル: TECH_STACK（PRDまたは独立ドキュメント）
