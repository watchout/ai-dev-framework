# Knowledge Database Index - 知識データベース

> Discovery/Businessフェーズで参照する外部知識・トレンド情報

## 概要

このディレクトリには、プロダクト開発の意思決定に役立つ外部情報を格納します。
ディスカバリーフェーズや合議において、根拠として参照されます。

## ディレクトリ構造

```
docs/knowledge/
├── _INDEX.md           ← このファイル
├── trends/             ← 業界トレンド・技術動向
│   └── CES_2026_ANALYSIS.md
├── competitors/        ← 競合分析データ（プロジェクト固有）
├── market/             ← 市場調査データ
└── domain/             ← ドメイン固有知識
```

## 利用方法

### Discovery開始時

```markdown
1. docs/knowledge/ を確認
2. 関連する知識データを読み込む
3. 既知の情報は確認形式で質問
   「○○と理解していますが合っていますか？」
4. 提案時は根拠を示す
   「知識データ（CES_2026_ANALYSIS.md）によると…」
```

### 合議での参照

```markdown
## 合議: [議題]

### 根拠となる知識データ
- CES_2026_ANALYSIS.md: Agentic AIがトレンド
- [競合分析]: 市場の○○が課題

### 各専門家の見解
[知識データを踏まえた見解]
```

## 追加ルール

### 知識データの追加

1. 信頼できるソースからのみ追加
2. 更新日を明記
3. このINDEXを更新

### 矛盾発見時

ユーザーの回答と知識データに矛盾がある場合:
→ 「ユーザーの回答と知識データに矛盾があります」と報告

### 不足発見時

必要な情報が知識データにない場合:
→ 「この情報は知識データにありません。追加を推奨します」と報告
