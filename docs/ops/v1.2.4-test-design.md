## 1. デプロイ手順
### 1.1 前提条件
- ADF v1.2.0/1/2/3 完了 + dogfood
- `fast-check` dependency CEO approval 済
- macOS dev fleet (template / hook 前提)

### 1.2 手順
1. `git pull origin main`
2. `npm install` (fast-check 等)
3. `framework gate validate spec` で既 spec の Coverage Gap section retrofit (warn only mode から start)
4. `tests/acceptance/` / `tests/property/` 配置
5. `traceability.csv` 既 spec の retrofit (auto-generated 推奨)
6. PR で動作確認

### 1.3 デプロイ後確認
- 任意 PR で Coverage Gap 不在 → warn (or exit 2 strict mode)
- Property test 1000 runs pass
- trace verify exit 0

## 2. ロールバック手順
- coverage gap が dev 速度阻害 → warn only mode に切替
- Property test の seed flaky → seed 固定 or skip
- trace verify drift 連発 → traceability.csv 再生成

## 5. 障害対応 Runbook (3 症状以上)

### 5.1 症状: Property test flaky
- 一次対応: seed 固定で再現、retry 3 回 + counterexample 記録
- エスカレーション: 連続 3 commit 同症状で auditor mention
- 再発防止: seed 固定 + counterexample regression test 追加

### 5.2 症状: trace drift 連発
- 一次対応: traceability.csv 再生成、broken link list を Issue 化
- エスカレーション: 1 週間 5+ drift で CTO mention
- 再発防止: code refactor 時に trace 自動更新 hook (別 spec で扱う)

### 5.3 症状: Coverage Gap section 形式違反
- 一次対応: parser regex の verify、template format 確認
- エスカレーション: 同症状 3 件で template 改訂 PR
- 再発防止: parser unit test + template visible warning

### 5.4 症状: fast-check resource 過多 (CI runtime 超過)
- 一次対応: ADF_PROPERTY_RUNS を 500 に削減 (env override)
- エスカレーション: 連続 timeout で CTO mention
- 再発防止: heavy invariant を nightly に分離

- 四半期: trace.csv 整備、orphan link cleanup
