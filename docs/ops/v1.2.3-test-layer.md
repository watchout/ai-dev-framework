## 1. デプロイ手順 [必須]

### 1.1 前提条件
- ADF v1.2.0/1/2 完了
- agent-comms-mcp の state-daemon merged
- macOS dev fleet (launchd 前提)

### 1.2 手順
1. `git pull origin main`
2. `npm install` (test runner deps)
3. `bash tests/e2e/launchd-integration.test.sh` 実機 verify
4. PR template (4-Evidence section) を repo に配置

### 1.3 デプロイ後確認
- E2E test pass
- 4-Evidence routine が CTO L3 で稼働

## 2. ロールバック手順 [必須]
- E2E test failure 連発 → CI workflow から E2E job を一時 disable
- 4-Evidence routine が dev 速度阻害 → CTO 判断で warning only mode

## 3. 監視項目 [必須]

| メトリクス | 正常範囲 | アラート | 通知先 |
|---|---|---|---|
| E2E test pass rate | > 95% | < 90% | #dev-adf |
| 4-Evidence 欠如 PR | 0/week | > 0 | #dev-adf |

## 4. SLO [必須]
- E2E test pass rate > 95%
- 4-Evidence 強制率 100% (CI gate)

## 5. 障害対応 Runbook [必須、3症状以上]

### 5.1 症状: E2E test flaky 連発
- 一次対応: retry log 確認、test fixture の race condition 調査
- エスカレーション: 同症状 5+ 件で CTO mention

### 5.2 症状: launchd test timeout
- 一次対応: ThrottleInterval 確認、process lock 調査
- エスカレーション: 連続 3 commit で CTO + agent-com-dev mention

### 5.3 症状: 4-Evidence section 欠如 PR が頻発
- 一次対応: PR template 配置確認、CI gate 動作確認
- エスカレーション: 1 週間 5+ 件で lead-bot review prompt 強化

## 6. 定期メンテナンス
- 四半期: E2E test runtime / coverage review
- 月次: 4-Evidence 形式 / regex pattern review
