# SPEC: ADF v1.2.2 — 高度 hook + Plan Mode + Verify 強化

## 0. メタ
- 作成日: 2026-05-08
- 関連 ADR: ADF v1.2.0 (4 層体系)、SPEC-DOC4L-008 (v1.2.1 Hooks 基盤)
- 起点 proposal: `proposals/v1.2.x/adf-v1.2.x-spec-proposal.md` SPEC-DOC4L-009
- Honesty: 全 claim に [検証済] / [文献確認] / [推測] ラベル必須

## 1. 目的 (Goals) [必須]

v1.2.1 の基本 hook を超えて、4 bot ヒアリングで発見した **深層 failure mode** (label gaming / citation hallucinate / drift 累積 / 推測で答える) に対応。加えて Boris 原則 1「Plan Mode で調査と実装を分ける」を ADF 配下で仕組み化し、CTO 自認の「spec 全文未読 dispatch」failure を構造的に防止。

## 2. 非目的 (Non-goals) [必須]

- v1.2.1 で扱う基本 4 種 hook の改造は本 spec の範囲外 (拡張は §3.2 / §3.3 で記述)
- Plan Mode の Claude Code 公式仕様変更は out of scope
- bot の prompt rule (advisory) の改訂は本 spec で扱わない
- agent-com fleet 全体への展開は別 task (§5 期待効果に記述、運用判断)

## 3. ユーザーストーリー [必須]

- **As CTO**、`framework dispatch <task> --mode=implement` 起動前に直近 24h 内の Plan ファイル存在を hook が check するので、spec 未読 dispatch が技術的に発生しない。
- **As auditor**、bot 出力に [検証済] / [文献確認] / [推測] のいずれも付かない断定文があれば Stop hook が block する → label gaming 防止。
- **As lead-bot**、`[文献確認 file:line] "..."` 形式の citation を出した bot が hallucinate していれば mechanical grep で検出 + block される。
- **As CEO**、main merge 後に GitHub Actions が `framework verify post-merge` を強制実行、全方位検証 pass まで「verified」状態に進まない。
- **As 新 session の bot**、SessionStart で過去 verdict (audit-pass / audit-block / spec-decision / drift-found) が auto-load され、前任 verdict と矛盾しない判断ができる。

## 4. 機能要件 (Core) [必須]

### 4.1 [SPEC-DOC4L-009-FR-001] Plan Mode 必須化 (F1)

#### 4.1.1 F1-1: SessionStart hook 拡張

`session_type=implementation` の場合、`.framework/plans/latest.md` 存在 check (24h 以内 mtime)。なければ `additionalContext` に "Plan Mode 必須" error message を出して exit 1 (実装 session 進行 block)。

#### 4.1.2 F1-2: framework dispatch コマンド (新設)

```
framework dispatch <task-id> <bot> --mode=plan
  → Plan Mode session 起動、bot に「調査せよ、実装するな」prompt
  → output を .framework/plans/<task-id>.md に保存

framework dispatch <task-id> <bot> --mode=implement
  → 直近 Plan ファイル存在 + 24h 以内を check
  → なければ block + plan mode 起動を suggest
  → あれば実装 session 起動、Plan を context に注入
```

### 4.2 [SPEC-DOC4L-009-FR-002] label check hook (F2)

Stop hook 拡張。bot 直前 output を transcript log から取得、断定 pattern (動く / 動かない / 必要 / 不要 / 対応 等) を regex 抽出、各断定に [検証済] / [文献確認] / [推測] ラベルが付いているか mechanical 検証。違反時 `{"decision":"block","reason":"Label missing on technical assertions: <list>"}`。

### 4.3 [SPEC-DOC4L-009-FR-003] citation 検証 hook (F3)

label-check.sh 拡張。`[文献確認 file:line(-line)] "..."` 形式の citation を regex 抽出、file:line を sed/awk で読み出し、quoted_text と string 比較。不一致なら `{"decision":"block","reason":"FAKE_CITATION: <file:line> does not contain '<quoted>'"}`。

- 想定: file 不在 / line 範囲外時は **fail-open** (verification skip + audit log 記録)。block しない (= bot は continue)、ただし audit log 経由で operator visibility 確保。

### 4.4 [SPEC-DOC4L-009-FR-004] post-merge full verify (F4)

GitHub Actions workflow `.github/workflows/post-merge-verify.yml` で `push` to `main` 時に `framework verify post-merge --commit ${{ github.sha }}` を実行:

- 全 test suite 実行
- DB invariant check (project 別、`.framework/verify/db-invariants.sql` で定義)
- spec ↔ test mapping completeness check
- branch protection drift check
- 結果を PR comment + Discord に post
- 全 pass 時のみ ADF state を `verified` に更新 (`.framework/verify/state.json`)

bot は `framework verify status --commit X` で verified 状態を確認しないと「完了」発言不可 (label-check.sh が「完了」発言時に verified state を強制 check)。

### 4.5 [SPEC-DOC4L-009-FR-005] verdict persist 機構 (F5)

```
framework verdict log <bot> <verdict-type> <content>
  → .framework/verdicts/<bot>/<timestamp>-<type>.md に保存
  → MEMORY.md に link を append (path のみ、本文は別 file)

framework verdict recent <bot> --limit 10
  → 直近 N 件の link を取得
  → SessionStart hook (F1-1 拡張) で auto-load
```

verdict_type: `audit-pass` / `audit-block` / `spec-decision` / `governance-violation-detected` / `drift-found` / `dispatch-completed`。

## 5. インターフェース (Contract) [必須]

### 5.1 CLI 契約

```
framework dispatch <task-id> <bot> --mode=plan
  exit 0: plan 完了、.framework/plans/<task-id>.md に書込み
  exit 2: plan failure (bot が implement に踏み込んだ等)

framework dispatch <task-id> <bot> --mode=implement
  exit 0: 実装 session 起動成功
  exit 2: Plan ファイル不在 / 24h 超過、suggestion を stderr 出力

framework verify post-merge --commit <sha>
  exit 0: 全 pass、.framework/verify/state.json を verified に更新
  exit 2: 1 つ以上 fail、PR comment + Discord post

framework verify status --commit <sha>
  exit 0: verified
  exit 2: not-verified

framework verdict log <bot> <type> <content>
  exit 0: 書込み成功

framework verdict recent <bot> --limit <n>
  exit 0: stdout に link list (改行区切り)
```

### 5.2 hook script 契約

- label check: Stop hook、JSON output で `decision: block` 可
- citation 検証: 同 Stop hook 内で実行、file IO + 文字列比較

### 5.3 イベント / メッセージ [該当時]

- post-merge verify 結果は Discord channel に webhook で投稿 (channel id は project 別 config)

## 6. 非機能要件 (Detail) [必須]

### 6.1 性能

- label check < 2 秒 (transcript 1000 行想定)
- citation 検証 < 3 秒 (citation 10 件 / output)
- `framework verify post-merge` は GitHub Actions 内 < 10 分 (test suite 9 分 + α)

### 6.2 可用性 (SLO)

- Plan ファイル不在で本来許可すべき緊急 hotfix が block されないこと: emergency override env (`ADF_PLAN_BYPASS=1`) を提供、使用は audit log + CTO escalate
- post-merge verify failure 時、main は revert 可能な状態を維持 (verify は post-hoc、merge gate は L3/L4 の責務)

### 6.3 セキュリティ要件 [app/api プロファイルで必須]

#### 6.3.1 脅威モデル (STRIDE)

| カテゴリ | 該当内容 |
|---|---|
| Spoofing | bot identity 偽装は範囲外 |
| Tampering | verdict log の改竄 → git で commit 必須 (.framework/verdicts/ は SoT)、PR review |
| Repudiation | verdict log は append-only、削除 PR は L3 review |
| Information Disclosure | verdict 本文は project 内に留める、Discord post は summary のみ |
| Denial of Service | label check の正規表現 catastrophic backtracking を回避 (RE2 推奨) |
| Elevation of Privilege | `ADF_PLAN_BYPASS=1` 使用は監査 log + CTO escalate、放置不可 |

#### 6.3.2 OWASP Top 10:2021 マッピング

- A04 Insecure Design: emergency override は audit + escalate 経由のみ
- A08 Integrity Failures: verdict / Plan ファイルは git 管理 + commit 必須
- A09 Logging Failures: 全 dispatch / verify / verdict を `.framework/` 配下に log
- 他: N/A

#### 6.3.3 データ分類

- Plan ファイル: [x] 公開 (repo 内、PR review 対象)
- verdict ファイル: [x] 公開 (repo 内)
- transcript log (label check 入力): [ ] 機密 (LLM session 履歴、project 内に留める、外部送信禁止)

### 6.4 監査ログ要件 [該当時]

- `framework dispatch` / `framework verify` / `framework verdict` の全実行を `.framework/audit/{date}.jsonl` に記録
- `ADF_PLAN_BYPASS=1` 使用時は `audit/bypass-{date}.log` に別 file 出力 + Discord post (CTO mention)

## 7. 受入基準 (Acceptance Criteria) [必須・Gherkin形式]

### 7.1 [SPEC-DOC4L-009-FR-001] Plan Mode 必須化

```gherkin
Feature: Plan Mode 必須
  Scenario: Plan ファイルなしで implement 起動 → block
    Given .framework/plans/ に直近 24h 内の plan ファイルなし
    When framework dispatch task-1 bot-x --mode=implement を実行
    Then exit code は 2
    And stderr に "Plan Mode 必須、--mode=plan で先に調査せよ" を表示

  Scenario: Plan あり → implement 通過
    Given .framework/plans/task-1.md の mtime が 1h 前
    When framework dispatch task-1 bot-x --mode=implement を実行
    Then exit code は 0
    And bot session に Plan 内容が context として注入される
```

### 7.2 [SPEC-DOC4L-009-FR-002] label check

```gherkin
Feature: label check hook
  Scenario: label なし断定 → block
    Given bot output に "これは動く" (label なし)
    When Stop hook が発火
    Then JSON output decision=block
    And reason に "Label missing" を含む
```

### 7.3 [SPEC-DOC4L-009-FR-003] citation 検証


### 7.4 [SPEC-DOC4L-009-FR-004] post-merge verify

```gherkin
Feature: post-merge verify

    And test suite + DB invariant + trace check が走る
    And 全 pass で .framework/verify/state.json が verified=true に更新される

```

### 7.5 [SPEC-DOC4L-009-FR-005] verdict persist


## 9. リスクと緩和策 [該当時]

| リスク | 緩和策 |
|---|---|
| Plan Mode 必須化で hotfix が block | `ADF_PLAN_BYPASS=1` emergency override + audit log + CTO escalate |
| label check の false positive (引用文中の断定を誤検知) | regex tuning + dogfooding 期間で false positive rate < 5% を確認 |
| citation 検証の file path mismatch (relative vs absolute) | `$CLAUDE_PROJECT_DIR` 起点の正規化処理を hook script に組込 |
| post-merge verify が長時間化、developer feedback 遅延 | GH Actions timeout 10 分、超過は warn (block ではない、verified=pending 状態) |
| verdict log が disk 圧迫 | 90 日 rotate を OPS で定義 |
| script 制御 vs Hook 使い分けの誤解 | OPS-DOC4L-009 §「制御機構の使い分け原則」+ Notion 共有 doc 参照 |
