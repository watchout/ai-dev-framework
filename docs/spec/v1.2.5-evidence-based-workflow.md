[文献確認 ARC 本日 honest 反省累積 7+ 件、CEO directive `5d04c953`]:

「LLM agent に対して、必ず実 file (spec/impl/実装 file) を読み、実 DB / log + Web 検索を元に常に **証拠がある形で全工程**を行う」 を script 制御として ADF に組込む。

本日 ARC が累積した「impl/schema 実態を verify せず spec 起草」反省 patterns (`feedback_check_ssot_before_drafting` 累積 7+ 件) の **構造的 fix**。

## 2. 非目的 (Non-goals) [必須]

- v1.2.2 F2/F3 (label/citation check) の置換ではなく **補完**

## 3. ユーザーストーリー [必須]

- **As 全 dev-bot**、PR 起票時に `[検証済]` 断定を書く前に file 引用 / DB query / log / URL を必ず提出する script 強制で、想定だけの起票が技術的に block される
- **As ARC**、spec 起草時に impl 実態を grep していない場合、§Evidence section 不在で CI gate に block されて再起草を強制される
- **As lead-bot / auditor**、PR review 時に §Evidence section の実在 / 一致を `framework evidence verify` で機械検証可能、想定だけの review を回避
- **As CTO / CEO**、merge 候補の §Evidence が再実行で broken していないか CLI で確認可能

## 4. 機能要件 (Core) [必須]

### 4.1 [SPEC-DOC4L-016-FR-001] §Evidence section 必須化 (F1)

全 SPEC (`docs/spec/*.md`) と PR description に §Evidence section を必須化:

```markdown
## §Evidence (本 spec / PR の主張根拠)

### 実 file 引用
- `path/to/file.ts:42-50` (claim X の根拠) [content quoted]

### 実 DB query 出力
- `psql -c "SELECT ..."`:
  ```
  (output)
  ```

### 実 log 抜粋
- `tail -N /path/to/log`:
  ```
  (output)
  ```

### Web 検索 / 公式 doc URL
- https://... (claim Y の根拠)

### [検証済] ラベル付き断定の根拠紐付け
- claim X → 実 file 引用 §
- claim Y → Web URL §
```

### 4.2 [SPEC-DOC4L-016-FR-002] CI gate (F2)

`.github/workflows/evidence-check.yml` で:
- §Evidence section 不在 PR → exit 2 (block)
- 4 sub-section 全空 → warn (small PR 許容)
- `[検証済]` 断定 1+ あって §Evidence 空 → exit 2 (block、honesty 違反)

### 4.3 [SPEC-DOC4L-016-FR-003] Pre-commit hook (F3)

`.claude/hooks/pre-commit-evidence-check.sh`:
- commit message + diff から `[検証済]` 断定抽出
- 各断定が file 引用 / DB query / log / URL を持つか mechanical check
- 不一致 → exit 2 (block)

= Hook 不可避 case 1 (tool 呼出 BLOCK) 該当。

### 4.4 [SPEC-DOC4L-016-FR-004] `framework evidence verify` CLI (F4)

```
framework evidence verify <pr-number>
  exit 0: 全 §Evidence claim が file 実在 / line range 一致 / DB query 再実行可能 / URL 200 OK
  exit 2: 1+ claim で証跡 broken
```

### 4.5 [SPEC-DOC4L-016-FR-005] PR template 強化 (F5)

`.github/PULL_REQUEST_TEMPLATE.md` に §Evidence 必須 section default 含める。

### 4.6 [SPEC-DOC4L-016-FR-006] 違反時 rollback (F6)

- §Evidence 不在 → CI block、author 追記
- `[検証済]` + §Evidence 空 → CI block (honesty)
- false evidence (file grep 不一致 / URL 404) → review reject、ARC patch 要請

## 6. 非機能要件 (Detail) [必須]

### 6.1 性能
- `framework evidence verify` < 10 秒 (中規模 PR、claim 10 件想定)
- pre-commit hook < 3 秒 (claim 5 件想定)

### 6.2 可用性
- evidence verify は GitHub API + git grep + file IO + curl で実装、外部依存 minimum

### 6.3 セキュリティ要件 [app/api プロファイルで必須]

#### 6.3.1 STRIDE
| カテゴリ | 該当内容 |
|---|---|
| Spoofing | N/A |
| Tampering | §Evidence 改竄 → git 管理 + PR review |
| Repudiation | 全 evidence verify を CI log + PR comment に永続化 |
| Information Disclosure | §Evidence に PII / secret 含めない (filter 必須) |
| DoS | URL 200 OK check の retry 制限 (3 回) |
| Privilege Escalation | N/A |

#### 6.3.2 OWASP: A04 Insecure Design (本 spec で structural 解消)、A09 Logging Failures (evidence audit で対応)

#### 6.3.3 データ分類
- §Evidence: [x] 公開 (PR description)
- DB query 出力: filter 必須 (PII 除去)
- log 抜粋: filter 必須 (secret 除去)

### 6.4 監査ログ要件
全 `framework evidence verify` 実行を CI log に永続化、`framework verify post-merge` (v1.2.2 F4) と連携。

    Given PR description に "[検証済] X が動く"
    And §Evidence section 全空

```gherkin

  Scenario: commit message に [検証済] without file 引用
    Given commit message "feat: X [検証済]"
    And diff に file 引用 / DB query なし
    When pre-commit hook trigger
    Then exit 2 (commit block)
```

## 9. リスクと緩和策 [該当時]

| リスク | 緩和策 |
|---|---|
| §Evidence 必須化が dev 速度を落とす | small PR は warn のみ (block しない)、large PR / `[検証済]` 含む PR のみ block |
| false negative (CI が evidence 形式 spoof を検出できない) | reviewer (lead/auditor) の手動 spot-check + `framework evidence verify` re-execution |
| URL 200 OK check の flaky | retry 3 回 + warn (block しない) |
| pre-commit hook が dev 体験を悪化 | `--no-verify` で bypass 可能 (audit log に記録、bypass-detection.yml で merge gate) |

## 10. 制御機構選定原則 [必須]

[文献確認: Notion canonical https://www.notion.so/35ad2b26f3dc8122b9f5e513b769d4e4]

| FR | 機構 | 不可避 case 該当 | 根拠 |
|---|---|---|---|
| FR-001 §Evidence section | doc 規約 (PR template + spec template) | — | 配布 template = repo file |
| FR-002 CI gate | **script** (GitHub Actions workflow) | — | merge gate は script default |
| FR-003 Pre-commit hook | **Hook** (case 1 = tool BLOCK) | case 1 (commit tool 呼出 block) | git commit に対する block は hook 不可避 (commit 後の script 制御では遅い) |
| FR-004 evidence verify CLI | **script** (CLI + GitHub API) | — | CLI は script default |
| FR-005 PR template | doc 規約 | — | 配布 template |
| FR-006 違反時 rollback | **script** (CI + review) | — | merge gate は script default |

= **FR-001/002/004/005/006 が script 制御**、FR-003 のみ **Hook 不可避 case 1** 該当。原則整合 ✅

### 10.3 違反時 rollback (本 spec)
script で代替可能なのに Hook で実装 → CTO L3 review で reject (本 spec FR-003 以外で Hook 採用は禁止)。
