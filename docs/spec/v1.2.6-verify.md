## §4 Regression discipline

### §4.1 新規 spec 追加時

- `docs/spec/v1.2.X-*.md` 追加 PR → CI `spec-audit` workflow trigger
- F1-F7 + meta_spec exception 適用、no-flag default (= §5.3.1 per-file 自動選別)
- 新規 spec は strict 対象、F1-F5 で 1+ fail → CI red → merge block

### §4.2 adapter / check 改変時

- `src/cli/commands/gate/**` 変更 → `spec-audit` workflow trigger paths 該当
- 15 件 全 PASS が merge gate
- 新規 check / port 拡張時は本 VERIFY の §2 inventory + §3 mapping を **PR 内で更新**、SPEC ↔ IMPL ↔ VERIFY drift 防止

### §4.3 corpus-v1 改変時

- `__fixtures__/corpus-v1/*` 変更 → Smoke A/B 期待値の re-verify
- corpus 拡張時は新 spec file + `git-history.json` 該当 path 追記 + integration test の per-file finding assertion 追加

## §5 Known coverage gaps

### §5.1 F5 semantic 比較 unit fixture 未追加

cycle 9 frozen 12 fixture には F5 unit fixture が含まれず、F5 の旧版 diff logic は adapter level (`fileAtRef` 戻り値 null で skip) でのみ unit verify。real git diff 経由の end-to-end verify は CI step (= 実 repo `docs/spec/**` audit) で indirect 検証。

follow-up: F5 unit fixture (fakeGit が `fileAtRef` で旧 content を返却、現 content と diff、§0 patch table cross-check) を v1.2.7+ で追加検討。

### §5.2 F6 Notion link unit fixture 未追加

同様に F6 unit fixture も cycle 9 frozen 範囲外。`makeFakeLink` の probes record 経由で fixture 追加可能、follow-up 候補。

### §5.3 meta_spec frontmatter regex の末尾 comment 拒否 bug (= Task 3 hotfix 予定)

`adapters/spec-fs.ts` `parseFrontmatterImpl()` の `meta_spec` 検出 regex `/^meta_spec:\s*true\s*$/m` は **末尾 inline comment を拒否**。

```yaml
meta_spec: true   # cycle 2 patch、Gap 1 解消 anchor
# ↑ 末尾 comment ありで regex match せず、metaSpec=false 扱い
```

現状 `docs/spec/v1.2.6-spec-audit-gate.md` の frontmatter は末尾 comment ありで、CLI run 時に F2 exception が適用されず §12-§14 欠如で fail する **bug** (CEO `c46cbc2d` directive コンテキストで identified)。

**対応**:

- v1.2.7 Task 3 (CEO bug 判断待ち) で regex 修正候補:
  - (a) `/^meta_spec:\s*true(?:\s|#|$)/m` (trailing space / `#` / EOL 許容)
  - (b) YAML parser 経由 (新規 dep `js-yaml` 等、依存追加判断必要)
  - (c) frontmatter 側で comment 削除 (spec doc 1 行修正)

本 VERIFY layer では Known gap として明示記録、Task 3 解消後に本 §5.3 を削除予定。

### §5.4 git-less 環境未 test

`adapters/git-history.ts` は `node:child_process.execFile('git', ...)` 依存、git 不在環境では catch して null/false fallback。non-git 環境 (= git 未 install / `.git/` 不在) での挙動は unit fixture では skip 系のみ、production failure scenario は未網羅。
