## 4. 機能要件 (Core) [必須]

### 4.1 [SPEC-DOC4L-017-FR-001] SPEC ID 重複 check (F1)
`framework gate validate spec --check=id-uniqueness`
- 全 `docs/spec/*.md` の frontmatter `id:` を grep
- 重複 → exit 2 (block)

### 4.2 [SPEC-DOC4L-017-FR-002] 4 layer template 準拠 check (F2)
`--check=template-compliance`
- SPEC / IMPL / VERIFY / OPS の必須 section を template と比較
- section 欠如 → exit 2

### 4.3 [SPEC-DOC4L-017-FR-003] 1 file = 1 spec ID 規約 (F3)
`--check=one-id-per-file`
- frontmatter `id:` 1 つだけ
- bundle 例外時は spec 内 「### Bundle 例外明示」 section 必須 (例: `SPEC-DOC4L-012-015`)
- 例外 section 不在で複数 ID → exit 2

### 4.4 [SPEC-DOC4L-017-FR-004] §10 制御機構選定原則 準拠 (F4)
`--check=control-mechanism`
- `## 10. 制御機構選定原則` section 存在
- FR per script/Hook 選定根拠記述
- Hook 採用時、不可避 4 case 明示
- 不在 / 違反 → exit 2

### 4.5 [SPEC-DOC4L-017-FR-005] backward compatibility check (F5)
`--check=backward-compat`
- 過去 version との semantic 比較
- silent breaking → exit 2
- 廃止は §0 patch table で明示必須

### 4.6 [SPEC-DOC4L-017-FR-006] Notion canonical link verify (F6)
`--check=notion-link`
- spec の Notion URL を HEAD 200 OK 確認
- `[文献確認: ...]` ラベル付帯確認
- 404 / ラベル不在 → warn (block ではない、URL rot 別 spec)

### 4.7 [SPEC-DOC4L-017-FR-007] completion criteria literal 化 (F7)
`--check=completion-literal`
- §「完了条件」 / §「Definition of Done」 から ambiguous 表現 (regex: `〜程度|〜ぐらい|大体|状況に応じて`) grep
- 検出 → warn (literal 化推奨)

## 5. インターフェース (Contract) [必須]

### 5.1 CLI 契約

```
framework gate validate spec [--check=<name>] [--strict]
  exit 0: 全 check pass
  exit 2: strict mode で 1+ fail
  --strict: warn を fail に昇格
  --check 指定なし: 全 7 check 実行

利用 example:
  framework gate validate spec --check=id-uniqueness
  framework gate validate spec --strict
```

### 5.2 CI gate 連携
- `.github/workflows/spec-audit.yml` で本 CLI を `--strict` で実行
- exit 2 → PR merge block

### 5.3 bootstrap (Phase 0 grandfather)
- 既存 spec は `--strict` ではなく `--bootstrap` mode で warn only
- 新規 spec (本 PR 以降) から `--strict` 適用
- 移行: 既 spec を順次 retrofit、各 retrofit PR で `--strict` 通過確認

## 7. 受入基準 (Acceptance Criteria) [必須・Gherkin形式]

### 7.1 [FR-001] ID 重複 → block
```gherkin
Given 2 spec file が同じ id: SPEC-FOO-001
When framework gate validate spec --check=id-uniqueness
Then exit 2 + "Duplicate ID: SPEC-FOO-001 in fileA, fileB"
```

### 7.2 [FR-002] template section 欠如 → block
```gherkin
Given SPEC.md に §1〜§6 のみ (§7-§9 欠如)
When framework gate validate spec --check=template-compliance
Then exit 2 + "Missing sections: §7, §8, §9"
```

### 7.3 [FR-003] bundle 例外明示なし → block
```gherkin
Given frontmatter id: [SPEC-FOO-001, SPEC-FOO-002]
And spec 内 "### Bundle 例外明示" section 不在
When framework gate validate spec --check=one-id-per-file
Then exit 2 + "Multiple IDs without exception declaration"
```

### 7.4 [FR-004] §10 不在 → block
```gherkin
Given SPEC.md に §10「制御機構選定原則」section 不在
When framework gate validate spec --check=control-mechanism
Then exit 2 + "Missing §10 制御機構選定原則"
```

### 7.5 [FR-005] silent breaking → block
```gherkin
Given v1.2.0 spec で FR-001 が定義
And v1.2.1 spec で FR-001 が削除 (§0 patch table 明示なし)
When framework gate validate spec --check=backward-compat
Then exit 2 + "Silent breaking: FR-001 deleted without §0 patch declaration"
```

### 7.6 [FR-006] Notion link 404 → warn
```gherkin
Given spec に "https://notion.so/dead-link"
When framework gate validate spec --check=notion-link
Then exit 0 (warn only)
And stderr に "Notion link 404: ..."
```

### 7.7 [FR-007] ambiguous DoD → warn
```gherkin
Given 完了条件 section に "テスト pass 程度"
When framework gate validate spec --check=completion-literal
Then exit 0 (warn only)
And stderr に "Ambiguous completion criteria"
```
