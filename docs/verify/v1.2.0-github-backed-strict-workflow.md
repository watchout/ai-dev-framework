# VERIFY: Shirube v1.2.0 Completion — GitHub-Backed Strict Workflow

## 0. 対応する SPEC
SPEC-DOC4L-018。

## 1. 検証方針
v1.2.0 completion は unit tests だけでは完了扱いにしない。以下を満たす必要がある。

- deterministic unit tests
- mocked GitHub integration tests
- live GitHub smoke where safe
- Shirube self-dogfood evidence
- kodama application evidence

## 2. Test matrix

| Area | Required tests |
|---|---|
| Adoption lifecycle | greenfield init/start, midstream retrofit/start, managed update/resume, managed update/force-start |
| GitHub readiness | no gh, no auth, no remote, non-GitHub remote, happy path |
| Role readiness | missing role, placeholder role, producer/authority exact collision, actor-label collision |
| Sequence validation | duplicate seq, missing seq, invalid format, stable sorted next |
| Sync | GitHub -> local import, local cache refresh, orphan detection |
| Resequence | dry-run, GitHub write, local write, idempotent rerun |
| Design change | SPEC impact blocks, IMPL-only warns, gate-design resumes |
| Update | concrete roles preserved, missing roles filled with placeholders |
| E2E | S1/S2/S3 lifecycle entry -> strict start/resume -> design -> implement -> gate -> review -> exit -> resume |

## 3. Acceptance commands
```bash
npm run type-check
npm run build:cli
npm run lint
npm test
npm audit --audit-level=high
npm run shirube -- roles doctor
npm run shirube -- start . --audit-level strict --force --dry-run
npm run shirube -- next --json
npm run shirube -- sync
npm run shirube -- resequence --dry-run
```

Commands that touch GitHub must support mocked executor tests and a documented live smoke.

## 4. Completion evidence
Before v1.2.0 complete:

- #169 has linked Shirube self-dogfood E2E evidence.

## 5. Failure policy
- Any strict GitHub readiness failure is BLOCK.
- Any duplicate/missing seq in autonomous mode is BLOCK.
- Any SPEC-impacting active design change without gate-design is BLOCK.
- Any update that overwrites concrete role binding is BLOCK.
- Any lifecycle path that marks a repository framework-led before `start` / `start --resume` / `start --force` is BLOCK.
