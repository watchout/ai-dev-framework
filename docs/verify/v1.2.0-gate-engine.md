## 3. PR C Required Tests
Workflow observability tests:

| Command | Required behavior |
|---------|-------------------|
| `shirube workflow status --json` | emits phase, profile, evidence summary, decisions, allowed/blocked actions |
| `shirube workflow doctor --json` | reports missing evidence and remediation; diagnostic only, exits 0 for workflow `BLOCK` decisions |
| `shirube workflow check --action <action> --json` | evaluates only action-scoped decisions and exits non-zero when scoped decisions cross `--fail-on` |
| `shirube workflow explain <rule-id> --json` | explains rule, profile, decision, evidence refs, remediation |
| local-only fixture | succeeds without GitHub auth |
| missing hearing fixture | explains required hearing/import/confirmation remediation |
| placeholder roles fixture | explains `shirube roles set ...` remediation |
| `draft_only` fixture | explains remote publish block |
| `implementation_start` action fixture | ignores remote-publish-only blocks |
| `remote_publish` action fixture | fails on remote publish blocks |
| missing `--action` fixture | fails explicitly rather than falling back to global BLOCK |

Output should be stable enough for MCP/GitHub wrappers. Avoid exposing private reasoning traces. PR C must not connect `workflow check` to `run`, hooks, CI, GitHub Checks, issue/PR generation, or design generation.

Initial action names:
- `design_draft`
- `implementation_start`
- `implementation_split`
- `remote_publish`
- `merge`
- `release`
