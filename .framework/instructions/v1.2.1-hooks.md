## 1. Interface contract (凍結)

### 1.1 sub-PR A — `framework hook init`

```ts
// src/cli/commands/hook/init.ts
export async function hookInit(opts: { force?: boolean; rollback?: boolean }): Promise<void>;
//   exit 0: 成功 (新規生成 or merge)
//   exit 2: 既存 settings.json と key 衝突、--force なし
//   exit 1: その他 error (lock 衝突 / fs error)
```

```ts
// src/lib/hooks/types.ts (sub-PR A 必須)
export interface HookEntry {
  matcher?: string;            // "Bash" / "Write|Edit|MultiEdit" / "startup|resume|compact"
  hooks: Array<{
    type: 'command';
    command: string;            // $CLAUDE_PROJECT_DIR/... prefix 必須
  }>;
}

export interface HookSettings {
  hooks: {
    PostToolUse?: HookEntry[];
    Stop?: HookEntry[];
    SessionStart?: HookEntry[];
    PreToolUse?: HookEntry[];
  };
}
```

```ts
// src/lib/hooks/settings-merger.ts
export function mergeHookSettings(
  existing: HookSettings | null,
  template: HookSettings,
  opts: { force?: boolean }
): { merged: HookSettings; conflicts: string[] };
//   existing が null の場合 template そのまま返却
//   key 衝突あり + force=false → conflicts 配列 non-empty で返す (caller が SettingsConflictError throw)
```

```ts
// src/lib/hooks/log.ts
export function appendHookLog(entry: HookLogEntry): Promise<void>;
//   .framework/hook-log/{YYYY-MM-DD}.jsonl に追記
//   write fail → stderr warn、bot session block しない (graceful、SPEC §6.2 fail-safe)
```

```ts
// src/lib/hooks/types.ts
export interface HookLogEntry {
  ts: string;                  // ISO 8601
  event: 'PreToolUse' | 'PostToolUse' | 'Stop' | 'SessionStart';
  matcher: string | null;
  command: string;
  exit_code: number;
  duration_ms: number;
  reason: string | null;       // exit 2 時の stderr first line
}
```

#### shell script 契約 (sub-PR A: `block-dangerous.sh`)

- stdin: Claude Code PreToolUse JSON payload
- exit code: `0` = pass、`2` = block (stderr に reason)
- block pattern (SPEC §4.4 F4-4 / Verify §3 異常系):
  - `gh pr merge .* --admin`
  - `git push (-f )?origin main$` (直 push)
  - `rm -rf( /| / )` (root 削除)
  - `DROP TABLE` / `DROP DATABASE`
  - `chmod 0?777`
- `$CLAUDE_PROJECT_DIR` prefix 経由で portable

### 1.2 sub-PR B — `stop-verify.sh` 契約

- stdin: Claude Code Stop JSON payload
- `stop_hook_active=true` → exit 0 (無限 loop 防止)
- それ以外: `npm test` (or 同等 test runner) を実行
  - pass → exit 0
  - fail → **stderr に reason JSON `{"decision":"block","reason":"Test suite must pass before completion"}` を出力 + exit 2** (SPEC §7.3 acceptance + auditor Pre-impl gate `60204409` case A 採択、Claude Code Hook native block 機構整合)
- 環境変数 `ADF_HOOK_DRY_RUN=1` で log only mode (block せず exit 0)

> **Note**: VERIFY §3 異常系の現記述「exit 0 (JSON で block)」は SPEC §7.3 と drift。本 instruction は SPEC §7.3 を SoT として case A (exit 2) を採用。VERIFY §3 patch 要請を ARC に escalate 済 (本 instruction 改訂と並行、merge 前に同期必須)。

### 1.3 sub-PR C — `post-edit-verify.sh` / `inject-spec-context.sh` 契約

#### `post-edit-verify.sh` (F4-1)

- stdin: PostToolUse JSON
- `tool_input.file_path` を抽出 (jq or python -c)
- file 拡張子に応じて `prettier --check` / `eslint` / `tsc --noEmit` を実行
- 失敗時 exit 2 + stderr に file path + lint error
- dependency 不在時は `command -v` で graceful skip + warn log (Runbook §5.3 整合)

#### `inject-spec-context.sh` (F4-3)

- matcher: `startup|resume|compact`
- 直近 modified spec files 抽出 (`git log --since=24h --name-only -- 'docs/spec/*.md'`、open decisions: 時間窓 default 24h、env `ADF_SPEC_INJECT_WINDOW=Nh` で override)
- 関連 ID (`SPEC-* | IMPL-*`) を抽出 (regex grep)
- stdout に JSON `{"additionalContext":"## Recent spec changes\\n- ...\\n"}` 出力
- recent spec 不在時は空 additionalContext (graceful、Runbook §5.4 整合)

## 2. Required behavior (凍結)

各 sub-PR で以下不変条件を遵守:

### 2.1 [全 sub-PR]

- **R-1**: hook 実行時間 < 5s (per event、SPEC §6.1)。超過時は warn log のみ、block しない
- **R-2**: hook script crash で bot session を block しない (SPEC §6.2 fail-safe)。ただし PreToolUse の `exit 2` (block) は意図的 block なので **R-2 と区別** (crash ≠ intentional block)
- **R-3**: 全 hook 実行を `.framework/hook-log/{date}.jsonl` に append (SPEC §6.4 + IMPL §6.1)
- **R-4**: hook script source は repo 内のみ参照、外部 fetch 禁止 (IMPL §8、template default)
- **R-5**: hook log は PII 含まない (`command` 全文のみ、stdin/stdout 記録禁止 — IMPL §8)
- **R-6** [SPEC §4.5 FR-005 + IMPL §1.1 + Claude Code 公式仕様]: 全 file path は forward slash、Windows native path 禁止 (`$CLAUDE_PROJECT_DIR` prefix での portable path 維持)

### 2.2 sub-PR A 固有

- **R-A1**: `framework hook init` は `--force` なしで既存 settings 衝突 → exit 2、`SettingsConflictError` を caller に伝播
- **R-A2**: init 前に `.claude/settings.json.bak.{ISO8601}` バックアップ作成 (IMPL §3.2 トランザクション境界)、merge 失敗時 finally で復元
- **R-A3**: `block-dangerous.sh` の block pattern は **hard-coded** (project 配下 fork で override 可、PR review 経由のみ — IMPL §8)
- **R-A4**: `framework hook init` 並行起動は `.framework/hook-init.lock` で排他制御、取得失敗 → exit 1 + `LockTimeoutError`

### 2.3 sub-PR B 固有

- **R-B1**: `stop_hook_active=true` 検出時は **必ず exit 0** (無限 loop 防止、SPEC §4.4 F4-2)
- **R-B2**: test runner command は `npm test` を default、project の `package.json scripts.test` を参照 (project 別の test command 差は env `ADF_STOP_TEST_CMD` で override 可、open decisions §5)
- **R-B3** [SPEC §7.3 acceptance + auditor `60204409` case A]: test fail 時の output は **stderr に JSON `{"decision":"block","reason":"..."}` を出力 + exit 2** (Claude Code Hook native block 機構整合、VERIFY §3 patch を ARC に escalate)

### 2.4 sub-PR C 固有

- **R-C1**: `post-edit-verify.sh` は dependency (prettier / eslint / tsc) 不在で graceful skip + warn (R-2、Runbook §5.3 — 編集者を block しないこと)
- **R-C2** [SPEC §4.4 F4-3 + SPEC §5.2 + Claude Code SessionStart 仕様]: `inject-spec-context.sh` の出力は valid JSON、stdout 1 line で `{"additionalContext":"..."}` form 必須 (公式仕様、これ以外は bot context 注入に失敗)
- **R-C3**: SessionStart matcher 順序は `startup|resume|compact` 全 case 対応

### 2.5 sub-PR D1 固有 (FR-002)

- **R-D1** [SPEC §5.1 CLI 契約]: `framework hook validate` は `exit 2` を CI gate として使用可能、`exit 1` はその他 error (lock / fs)
- **R-D2** [IMPL §2.2 関数シグネチャ + IMPL §3.1 シーケンス独立性]: `validateHookSettings(s: HookSettings): HookValidationResult` は side-effect free、純関数 (file IO / network なし、unit test で input → output 完結)

### 2.6 sub-PR D2 固有 (FR-003)

- **R-D3** [IMPL §1.2 変更ファイル + IMPL §5.2 拡張する既存関数]: F3 `_template.md` 更新は **既存 section を破壊しない** (additive で「検証層の構造」章のみ追加、行頭 `## ` で挿入)
- **R-D4** [IMPL §1.2]: SPEC-INDEX.md には **SPEC-DOC4L-008 (v1.2.1) entry のみ** 追加 (v1.2.2 は Issue #127 範囲、scope exclusion §3.3 厳守)

## 3. Forbidden behavior (凍結、anti-patterns)

### 3.1 [全 sub-PR] 禁止事項

- **F-1**: `process.env.CLAUDE_PROJECT_DIR` を読まない hook script は禁止 (FR-005 violation、portable break)
- **F-2**: hook script 内で外部 URL fetch (`curl` / `wget` / `nc`) は **禁止** (IMPL §8 セキュリティ、Verify §6 攻撃ベクタ)
- **F-3**: `.claude/settings.json` を **Write tool で直接生成しない** (必ず merger logic 経由、conflict 検知漏れの過去 incident: #318 admin merge cumulative pattern と同じ「機械強制 skip」失敗 mode)
- **F-4**: hook log に stdin / stdout / 環境変数 dump は **禁止** (PII / secret leak、IMPL §8 R-5)
- **F-5**: SPEC / IMPL / VERIFY / OPS doc を sub-PR 内で **無断改変禁止** (spec drift、改変必要なら patch sub-PR を別建て、ARC review 経由)
- **F-6**: `any` 型使用、`console.log` production code 残置、テストなし PR (CLAUDE.md §禁止事項)

### 3.2 sub-PR 固有禁止

- **F-A1**: `block-dangerous.sh` の pattern を **stdin / env 経由で動的読込しない** (固定 pattern が SoT、bypass の温床になる過去 incident: agent-com #318 admin merge は文書 rule 不徹底が原因 = 動的設定の同類 risk)
- **F-A2**: `framework hook init` で既存 settings.json を `--force` なしに上書きしない (R-A1 強制、incident: hotel-kanri 過去 .env 上書きで recovery 6h と同根)
- **F-B1**: Stop hook の test runner を `bash -c "true"` 等の no-op で代替しない (DoD 偽 pass、proposal §「post-merge skip 95%+ 防止」根拠の毀損 — agent-com CTO 自認の post-merge skip 常習を構造的に再現)
- **F-C1**: PostToolUse hook で **edit ファイル以外** の lint を全件走査しない (5s 性能 budget 超過、R-1 violation)
- **F-D1-1** (sub-PR D1): `framework hook test` で **実機を破壊する command 実行** (`rm -rf` / `git push`) しない、dry-run pure であること
- **F-D1-2** (sub-PR D1): `framework hook validate --strict` の suspicious pattern 検出は **warn のみ** (exit 0)、`exit 2` で block しないこと (VERIFY §6 「PR review で block」整合、T-D3 fixture と一致)
- **F-D2-1** (sub-PR D2): doc-only PR で TS / shell コードを変更しないこと (F3 純粋に template + INDEX 同期、scope creep 防止)

### 3.3 scope exclusion

- **out-of-scope**: v1.2.2 範囲 (label check / citation 検証 / post-merge full verify / verdict persist / Plan Mode 必須化) は **本 sub-PR 群で実装しない**。v1.2.1 dogfood 完了 gating で別 Issue #127 dispatch 予定 (SPEC §2 Non-goals)
- **out-of-scope**: Claude Code hooks 仕様自体の拡張 (公式仕様遵守のみ、SPEC §2)
- **out-of-scope**: bot LLM の prompt rule (advisory CLAUDE.md) は本 spec 不対象 (script 制御原則 0 と直交、SPEC §2)

## 5. Open decisions (implementer 自由)
