# notes/ — 実装メモ・申し送り

開発中に発見した知見、判断理由、技術負債、改善案をタスク単位で記録する。

## 対象読者
- Gate Validator（品質検証時の追加コンテキスト）
- Reviewエージェント（レビュー時の背景理解）
- 次のfeature実装者（前回の判断理由の把握）

## ファイル命名規則
`{taskId}-{短い説明}.md`
例: `auth-middleware-setup.md`, `api-rate-limiting.md`

## 記録すべきもの
- 技術的判断の理由（なぜAではなくBを選んだか）
- 発見した技術負債（TODO、workaround、既知の制限）
- SSOTの曖昧さ・矛盾（どう解釈したか）
- 依存関係の注意点（バージョン制約、非推奨API）
- 後続タスクへの申し送り（「次にXをやるときはYに注意」）

## 記録しなくてよいもの
- コードの説明（コード自体が説明すべき）
- テスト結果（CI/CDが管理）
- Gate検出結果（.learnings/が管理）
